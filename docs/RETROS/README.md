# RETROS — twiceshy session retrospectives

One file per significant working session: `NNNN-kebab-title.md`. A retro records
**process** learnings (what the workflow got wrong/right), not product decisions
(those are ADRs). Each retro that yields a durable lesson **edits the constitution**
(docs/CONVENTIONS.md) with the lesson; the retro itself carries only the delta and
the story. A lesson that lives only in a retro is a lesson that will be re-learned.

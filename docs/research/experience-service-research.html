Experience-as-a-Service

Feeding hard-won engineering experience to LLM coding agents — research deep dive · 2026-06-12 · 5 research angles, ~60 sources fetched, 18 load-bearing claims adversarially verified (16 confirmed, 2 corrected). Full cited report: docs/EXPERIENCE_SERVICE_RESEARCH.md

§0The thesis

LLMs have answers; they don't have experience. Experience is project-, version- and time-shaped: "we tried X here, it broke Y, and here's the test that proves the fix." Serving such records back to agents measurably improves coding outcomes — if retrieval is precise, records are validated & fresh, and injection is deterministic rather than "hope the agent asks." This repo's REGRESSIONS.md (symptom → root cause → fix → guarding test) is already the right artifact — it just isn't delivered at decision time.

Today that knowledge only helps if the file happens to be in context. The gap isn't knowledge creation — it's delivery.

Frontier-lab convergence: Anthropic (memory tool, CLAUDE.md/auto-memory) and OpenAI (Codex ~/.codex/memories/ over AGENTS.md) both ship plain auditable files, not vector DBs — validating the markdown-first design.

§3The most important decision: the injection channel

Where most "memory MCP server" projects quietly fail. The evidence stack: even Anthropic's reference memory server ships a prompt forcing retrieval each chat; the When2Tool benchmark shows agents call tools indiscriminately and prompt steering miscalibrates both ways; CLAUDE.md rules are violated even when read; Cody/Greptile — products that must be reliable — inject retrieval-side, never waiting for the model to ask.

1 · Per-turn deterministic injection (hooks / middleware)
guaranteed presence, survives compaction
2 · Session-start injection
decays as context fills
3 · Static memory file (CLAUDE.md / AGENTS.md)
~advisory; framed "may or may not be relevant"
4 · Model-judged index (Skills / agent-requested rules)
efficient middle ground for nice-to-have
5 · Hoping the agent calls a tool
least reliable channel

The #1 failure mode of this very system: near-miss injection. A single semantically-related-but-wrong retrieved doc cut accuracy by up to ~25pp (SIGIR 2024 — random noise was harmless; related distractors hurt). Context-rot: all 18 tested models degrade with length, even one distractor hurts; mid-context placement is worst. An experience DB is near-miss-dense by construction → stack-fingerprint filters, relevance floor, applicability conditions inside each trap card.

Promotion pipeline — git gives it for free: a new experience record is a pull request (diffable, signed, revertable). Provenance on every record; quarantined records never enter the push channel. "Only my agent writes" is not a defense (MINJA) — the PR gate is the real boundary.

§8Evals: how you'll know it works

Template (ReasoningBank): same task suite, memory on/off, measure success and steps/tokens — experience should make agents faster even when success ties.

The eval to pioneer (verified gap): trap-avoidance regression evals. For each recorded trap, a small task that walks an agent toward it; score whether injection prevented the documented failure. REGRESSIONS.md is a ready-made seed set ("render my_var_name in agent prose", "make the budget row survive restart"). No published suite measures "did the agent avoid a previously recorded trap."

Instrument the live path: log every injection, mark confirmed_helpful → feeds Doctor 4. Treat vendor memory benchmarks (LoCoMo/DMR) as marketing until independently replicated — the Mem0↔Zep numbers swing 58–84% depending on whose methodology you accept.

Verification notes

Two adversarial verifier agents re-checked 18 load-bearing claims against primary sources: 16 confirmed, 2 corrected and folded in — (i) CodeRAG-Bench favors dense over BM25 for code (→ hybrid framing); (ii) Mem0-scored-Zep is ~66% (the ~58% belongs to a different leg of the dispute); plus precision fixes: ReasoningBank deltas are the Gemini-2.5-flash config; "Power of Noise" harm comes from related distractors; the 25–38% deprecated rate splits 70–90% vs 9–18% by prompt freshness; Qdrant's NFS ban lives in the installation guide; sqlite-vec gained alpha ANN in v0.1.10. Known soft spots: no Celeron embedding benchmarks exist; vendor-internal evals (Anthropic +39%) unreplicated.
